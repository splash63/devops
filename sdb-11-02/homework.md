# Домашнее задание к занятию "`Кеширование Redis/memcached`" - `Гайченков Евгений`

## Задание 1

* Высокая нагрузка на сервер. При большом количестве запросов к базе данных или API сервер может не справляться с нагрузкой. Кеширование позволяет хранить часто запрашиваемые данные в оперативной памяти, снижая нагрузку на основной сервер.

* Замедление работы приложения. Если приложение делает много запросов к базе данных, это может существенно замедлить его работу. Кеш позволяет хранить результаты запросов и возвращать их мгновенно при повторном обращении.

* Проблемы с масштабируемостью. При росте числа пользователей система может не справляться с нагрузкой. Кеширование помогает распределить нагрузку и обеспечить стабильную работу даже при резком увеличении трафика.

* Высокие затраты на обработку данных. Каждый запрос к базе данных или внешнему сервису требует ресурсов. Кеширование позволяет сократить количество таких запросов, экономя вычислительные ресурсы.

* Зависимость от внешних сервисов. Если приложение использует сторонние API, сбои или задержки в их работе могут повлиять на работу всей системы. Кеш позволяет временно хранить ответы от внешних сервисов и использовать их при недоступности источника.

* Оптимизация сложных вычислений. Результаты ресурсоемких вычислений можно сохранять в кеше и использовать повторно, не выполняя их заново при каждом запросе.

* Снижение затрат на инфраструктуру. Благодаря уменьшению нагрузки на основные компоненты системы можно сократить расходы на серверные мощности и другие ресурсы.

* Проблемы с производительностью при агрегации данных. При работе с большими объемами данных, требующих сложной агрегации, кеширование промежуточных результатов может значительно ускорить обработку.

* Оптимизация работы с статическим контентом. Изображения, стили, скрипты и другие статические файлы можно хранить в кеше, уменьшая время их загрузки для пользователей.

* Балансировка нагрузки в распределенных системах. В системах с несколькими серверами кеширование помогает равномерно распределять нагрузку и обеспечивать консистентность данных между узлами.
